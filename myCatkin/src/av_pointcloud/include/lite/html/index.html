Lite Libraries

0.7

This is a header only set of libraries that are light weight and independent. Each library is contained in one header file and can be copied and used on its own independently.

The following is the list of current Lite libraries:

  • lite xformat - A C++ printf/scanf style IO library
  • lite array - A C++ Multidimensional Array Library
  • signal library
  • xml library
Rationale
The following are the design goals of the lite libraries:
  • Each library should be a single, independent header file that can be copied and included in other projects. They should not depend on anything other than standard C++ libraries.
  • The syntax should be intuitive and should be what one would expect.
  • There should be minimal performace overhead.
  • The libraries should be general enough and fully extensibe. The should provide different layers of abstraction and allow for future extentions. If they lack a feature, they should provide the right infrastructure necessary to implement that feature as a seamless user extention or as a furture addition to the library.
